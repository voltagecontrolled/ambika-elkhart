# v4.0 Sprint Plan

Triage and execution plan for the [`ambika-elkhart 4.0`](https://github.com/voltagecontrolled/ambika-elkhart/milestone/1)
GitHub milestone. The milestone backlog is the source of truth for *what*;
this doc is the source of truth for *order, scope, and what's been cut*.

Each in-scope item is sized for a single fresh session: implement, build,
flash, exercise on hardware, update `docs/wiki/MANUAL.md`, commit. Future
sessions should open with "we're working on issue #N from the v4.0 sprint
plan" so the decisions captured here don't have to be re-derived.

---

## Triage decisions

### Punted to v5.0

- **#24 — swng on S7a does nothing.** Swing semantics on a polymetric
  sequencer where each track has its own CDIV is a real design question
  (apply at master clock? per-track? phase-locked to which division?). Not
  a regression, not blocking 4.0. Needs a planning doc before
  implementation.
- **#18 — Wavefolder waveform.** Voicecard is at ~80% flash. Adding a
  net-new oscillator algorithm at the cost of reserved CZ filter-sim slots
  is the wrong tradeoff for a release whose thesis is *percussive step
  sequencer*. Belongs alongside the post-4.0 BLND≥64 linear-FM /
  audio-rate-LFO oscillator-expansion work already on `BOARD.md`.
- **#8 — Mixer S6b cosmetic fixes.** Pure polish; no real friction in
  actual use. Zero v5.0 lock-in risk in deferring.

### Rescoped

- **#10 — Patch saving.** Original 8-slot patch save shrinks to a single
  preview snapshot. The full multi-slot UX (with naming, copy/paste
  submenu, voice/sequence cross-track copy) becomes a v5.0 child issue.
  Rationale: the real user pain is "lost my jam to a power cycle" — that
  is solved by one snapshot. Slot picking and naming are conveniences,
  not load-bearing, and shipping them now would commit to a file format
  that v5.0's expansion would have to migrate or version.

  **Locked scope of saved state:**
    - All 6 `SeqTrack`s (pattern, locks, shadow).
    - All 6 voice patches.
    - Mixer settings.
    - Transport settings (BPM, swing, `mrst`).

  **Explicitly excluded:** live LFO phase, playhead position, any other
  ephemeral runtime state.

  **File format:** raw struct dump. No magic header, no version byte. If
  and when the multi-slot expansion lands in v5.0, the format is
  redefined and old snapshots are no longer guaranteed to load. This is
  the explicit contract — the feature ships under "preview" framing.

  **UX:** new page on S8a in front of the existing firmware-update
  screen (which moves to S8b). S1 green = Save (overwrite confirm if
  file exists), S6 green = Load, S8 red = Exit. Spare pots/buttons
  unused. SD I/O reuses the existing FAT layer in the firmware-update
  neighborhood.

## Work order

Tiers run small → large. Each tier is a natural session boundary. The
MANUAL refresh stays last so it reflects the locked v4.0 surface.

### Tier 1 — Data / lockable cleanup

- **#14 — RATE/CDIV ratio display.** Replace `kCDivValues[] = {1,2,3,4,6,8,12,16}`
  with `{3,4,6,8,9,12,18,24}`; bump `kNumTicksPerStep` to 12 (lcm of 3
  and 4) so `1/4` and `3/4` land on integer tick counts. Display labels
  `1/4, 1/3, 1/2, 2/3, 3/4, 1/1, 3/2, 2/1` in `seq_track_page.cc` and
  `seq_steps_page.cc`. Audit other call sites for assumptions about the
  old base-6 unit.
- **#23 — Drop "glid", make `gtim` lockable.** Remove the `glid` knob
  role from S5a, move `gtim` to bot3 (leave bot4 empty for now), add
  `gtim` to the page-3 lockable list (joining freq/famt/pamt/wave/vamt).
  Touches `seq_track_page.cc` plus the lockable dispatch and
  `PatchAddrToSeqField` mapping.

### Tier 2 — Medium features

- **#22 — Hold-S7 + encoder transport-jump gesture.** Add chord
  detection in `Ui::Poll`. `SeqStepsPage`'s encoder handler must yield
  to the chord even while in sequencer mode. Cycles through Transport
  (`PAGE_MULTI`) and Global Mixer from any context.
- **#9 — `mrst` (Master Reset) on Transport S7a.** Pot 2 (top, third
  from left). Range: `off | 2..128` steps in undivided clock units; on
  reaching the count, all tracks reset. Default `off`. Touches the
  transport page UI and the global tick counter in `sequencer.cc`.
- **#25 — Track clear function on S6a bot3.** Pot selects mode:
  `clr locks` / `clr steps` / `clr notes` / `clr voice` / `clr all`.
  Long-press S6 (~800 ms) → S6 flashes red rapidly → tap to confirm.
  The display must show the selected mode unambiguously while S6 is
  flashing so the user can't misfire `clr all` thinking they're on
  `clr locks`. Touches `seq_track_page.cc`, the button handler, and
  per-mode clear ops on `SeqTrack`.

### Tier 3 — Snapshot save/load (rescoped #10)

- **#10 — Preview snapshot.** See the rescope block above for full
  contract. Implementation surface: new page sibling to
  `controller/ui_pages/os_info_page.cc`, raw-dump serializer for the
  locked scope, FAT write/read for one fixed filename
  (e.g. `SNAPSHOT.BIN`). Existing firmware-update screen moves from S8a
  to S8b.

### Tier 4 — Doc closer

- **#17 — `docs/wiki/MANUAL.md` refresh.** Full rewrite against the
  as-built v4.0 surface. Update incrementally as each issue above
  closes; the final sweep verifies and resolves any
  `<!-- VERIFY: ... -->` markers via a per-page hardware walk.
  Authoritative inputs: `CHANGELOG.md`, `docs/planning/sequencer.md`,
  `docs/planning/voice_envelopes.md`, `docs/planning/mixer.md`,
  cross-checked against `controller/ui_pages/`.

## Per-issue session shape

For each in-scope item, a session should:

1. Read the touched files.
2. Implement.
3. Build via `./toolchain/build-squeeze.sh {controller,voicecard}/makefile`.
4. Sanity-check flash/RAM with `./toolchain/size-squeeze.sh {controller,voicecard}`.
5. Bump `kSystemVersion` (controller: `controller/controller.h:37`;
   voicecard: `voicecard/voicecard.h:35`) so the OS Info page
   disambiguates slots on hardware.
6. Confirm with the operator which voice slot number(s) to flash before
   copying — multiple slots may be running different builds for
   comparison.
7. Copy artifacts to `firmware/latest/` (`AMBIKA.BIN` and/or `VOICE#.BIN`).
8. Update the relevant section of `docs/wiki/MANUAL.md`.
9. Stage specific files, commit with a single-line imperative message,
   no trailers.
10. Close the GitHub issue with a short link to the commit.

---

## Stop conditions

End a session and surface to the operator if:

- Context feels saturated; clarity drops on follow-up edits.
- A build or flash issue blocks progress and needs hardware verification.
- An issue's scope expands beyond what's captured here — surface for
  decision rather than improvising.
- Controller flash overflows on the snapshot serializer (#10) — surface
  the trade-off rather than improvising on scope.
